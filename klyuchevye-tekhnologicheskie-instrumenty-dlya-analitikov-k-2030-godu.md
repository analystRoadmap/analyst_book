## 1. AI и большие языковые модели (LLM)

AI уже стал базовым слоем профессии. По [IIBA Top 6 BA Trends 2026](https://www.iiba.org/business-analysis-blogs/top-6-business-analysis-trends-to-monitor-in-2026/) бизнес-анализ смещается от «сбора требований» к стратегической роли, а AI переходит из категории инструмента автоматизации в категорию **decision partner**.

### Как LLM меняют повседневную работу аналитика

| Задача аналитика | Без AI | С AI |
|-----------------|--------|------|
| Расшифровка встречи | Ручная сводка, заметки, потеря деталей | Speech-to-text + LLM + ручная валидация |
| Черновик требований | Пишется с нуля | Генерируется основа, аналитик уточняет ограничения и контекст |
| Поиск конфликтов в требованиях | Ручной ревью | LLM помогает искать дубли, пробелы, противоречия |
| SQL и аналитические запросы | Пишутся вручную | NL → SQL с обязательной проверкой аналитиком |
| Резюме длинного документа | 30–60 минут | За минуты, если есть хороший промпт и проверка результата |

{% hint style="info" %}
К 2030 году базовым навыком аналитика становится не только промпт-инженерия, но и **управление контекстом**: что можно передавать модели, где нужен RAG, где обязателен human review, а где результат можно запускать дальше по процессу автоматически.
{% endhint %}

## 2. Agentic AI: автономные агенты под контролем

Тренд 2026 года — переход от одиночных ассистентов к **агентным системам**, которые умеют планировать шаги, обращаться к API, использовать базы знаний, запускать код и доводить задачу до результата. Но зрелый подход — это не «дать AI всё и ждать магии», а проектировать управляемую автономию.

По [Deloitte Tech Trends 2026](https://www.deloitte.com/us/en/insights/topics/technology-management/tech-trends/2026/agentic-ai-strategy.html) агентный AI быстро развивается, но масштабирование зависит от governance, качества интеграций, risk control и понятной бизнес-ценности.

### Что такое AI-агент vs чатбот

| | Чатбот | AI-агент |
|---|--------|----------|
| **Принцип** | Вопрос → ответ | Цель → план → цепочка действий → результат |
| **Автономность** | Реактивный | Проактивный в заданных границах |
| **Инструменты** | Обычно текст и поиск | API, БД, код, workflow, документы |
| **Память** | Чаще сессионная | Долгосрочная/контекстная с внешними хранилищами |
| **Пример** | «Какой курс доллара?» | «Проверь заявки за неделю, выдели аномалии, подготовь отчёт и отправь руководителю» |

### Инструменты agentic AI

| Инструмент | Что даёт |
|-----------|----------|
| **[OpenAI Frontier](https://openai.com/index/introducing-openai-frontier/)** | Платформа для построения, развёртывания и управления AI-агентами в enterprise-контуре |
| **Claude Code** | Агентный режим для инженерных задач, файлов и кода |
| **LangGraph** | Управление многошаговыми агентными графами и human-in-the-loop |
| **CrewAI** | Оркестрация ролей и многоагентных сценариев |
| **[Yandex AI Studio](https://yandex.cloud/ru/docs/ai-studio)** | Платформа для AI-приложений и AI-агентов на базе генеративных моделей |
| **[GigaChat B2B](https://giga.chat/b2b)** | AI-сценарии для корпоративных кейсов, в том числе поиск по документам и генерация UAT-сценариев |

### Роль аналитика в мире agentic AI

Аналитик здесь отвечает не за «текст запроса», а за **границы автономии**:

- определяет, какие решения агент принимает сам, а какие эскалирует человеку;
- описывает бизнес-правила, ограничения и запреты;
- проектирует точки human-in-the-loop;
- задаёт критерии завершения и качества;
- продумывает failure scenarios, аудит и трассируемость действий.

{% hint style="warning" %}
Самая частая ошибка — пытаться внедрять AI-агентов поверх неописанных процессов и хаотичных интеграций. Агент усиливает зрелый процесс, но не заменяет отсутствие архитектуры и governance.
{% endhint %}

## 3. Process Mining: рентген для бизнес-процессов

Process Mining позволяет строить картину **реального процесса** по цифровым следам из ERP, CRM, BPM, сервис-десков и других систем. Это особенно важно для аналитика: вместо субъективного AS-IS по интервью появляется AS-IS по данным.

По данным [Сбера](https://www.sberbank.ru/ru/sberpress/all/article?blockID=1303&lang=ru&newsID=1bf3cc2e-8626-459b-b117-243829d7949c&regionID=58&type=NEWS), объём российского рынка Process Mining в 2024 году составил порядка **900 млн рублей**, а внедрение процессной аналитики планируют **46% российских компаний**.

### Как это работает

```text
Логи из ERP / CRM / BPM / Service Desk
        ↓
Process Mining платформа
        ↓
Автоматическое построение фактического AS-IS
        ↓
Выявление отклонений, задержек и узких мест
        ↓
Приоритизация улучшений и автоматизации
```

### Почему это важно для аналитика

| Подход | Интервью и воркшопы | Process Mining |
|--------|----------------------|----------------|
| Источник | Люди и наблюдения | Системные логи |
| Объективность | Субъективно | На основе данных |
| Скорость | Дни и недели | Часы и дни |
| Масштаб | Ограниченное число кейсов | Тысячи и миллионы экземпляров |
| Отклонения | Часто не видны | Выявляются автоматически |

## 4. BPM-платформы: нервная система организации

BPM-системы не просто рисуют схемы — они делают процессы **исполняемыми**. Это означает, что маршрут работы, правила, SLA, роли и интеграции становятся частью живой системы, а не картинкой в Confluence.

На международном рынке BPM всё чаще описывается как **orchestration**. [Camunda](https://camunda.com/) прямо говорит про agentic orchestration, а [Flowable](https://www.flowable.com/) — про workflow automation с AI agents и process orchestration.

### Ключевые BPM-платформы

| Платформа | Тип | Особенности | Где сильна |
|-----------|-----|------------|------------|
| **[Camunda](https://camunda.com/)** | Open source / enterprise | Оркестрация процессов, сервисов и AI-агентов | Enterprise, микросервисы, сложные workflows |
| **[Flowable](https://www.flowable.com/)** | Open source / enterprise | BPMN + CMMN + DMN + AI orchestration | Комплексные процессы и кейсы |
| **ProcessMaker** | Low-code BPM | Визуальная автоматизация процессов | Средний бизнес, быстрый старт |
| **Power Automate** | Cloud / low-code | Сильная интеграция с Microsoft 365 | Office-экосистема |
| **[ELMA365](https://elma365.com/ru/)** | Российская low-code BPM | Исполнимые бизнес-процессы, CRM, CSP, Service | Универсальный корпоративный стек |
| **[BPMSoft](https://bpmsoft.ru/)** | Российская low-code BPM | CRM + BPM + low-code автоматизация | Продажи, сервис, операционные процессы |
| **[Comindware](https://www.comindware.ru/platform/)** | Российская low-code BPM | Быстрое построение цифровой организации | Workflow, кейсы, внутренние приложения |

### Как меняется роль аналитика в BPM

К 2030 году аналитик всё меньше «рисует BPMN ради документа» и всё больше задаёт:

- цель процесса;
- условия запуска и завершения;
- правила ветвления и исключений;
- SLA и контрольные точки;
- роли и права;
- интеграции с внешними системами.

{% hint style="info" %}
Для аналитика ключевой сдвиг: от моделирования как описания — к моделированию как **контракту исполнения**.
{% endhint %}

## 5. RPA: роботы для рутины, а не замена архитектуре

RPA остаётся важным слоем автоматизации там, где процессы регулярны, правила детерминированы, а интеграция через API недоступна или слишком дорога. Но к 2030 году RPA всё чаще становится частью более широкой связки: **BPM + RPA + AI + Process Mining**.

### Где RPA особенно полезен

| Отрасль | Типичные задачи | Эффект |
|---------|-----------------|--------|
| **Банки** | KYC-проверки, сверки, комплаенс-отчёты | Ускорение и снижение числа ручных ошибок |
| **Страхование** | Обработка заявок и документов | Меньше ручного ввода и переключений между системами |
| **Логистика** | Таможенные формы, статусы заказов, сверка данных | Быстрее операционный цикл |
| **HR** | Онбординг, табели, отпускные процессы | Снижение нагрузки на HR-команды |

### Международный и российский стек RPA

| Инструмент | Особенности |
|-----------|------------|
| **UiPath** | Один из глобальных стандартов роботизации и intelligent automation |
| **Automation Anywhere** | Enterprise RPA и document automation |
| **Power Automate Desktop** | RPA в Microsoft-контуре |
| **[PIX RPA](https://pix.ru/)** | Российская экосистема роботизации и аналитики |
| **[Primo RPA](https://primo-rpa.ru/)** | Российская RPA-платформа с no-code, low-code и only-code режимами |

### Роль аналитика в RPA

- выявить кандидатов на роботизацию;
- оценить частоту, вариативность и ROI;
- описать бизнес-логику и исключения;
- предусмотреть точки контроля и fallback-процедуры;
- отслеживать, не пора ли заменить RPA на нормальную интеграцию или BPM.

{% hint style="warning" %}
RPA — не лекарство от плохой архитектуры. Если задача лучше решается через API, ESB или BPM, робот не должен быть решением по умолчанию.
{% endhint %}

## 7. iPaaS и ESB: интеграция как новая обязательная грамотность

Современная организация — это десятки приложений, сервисов, API и потоков событий. Поэтому аналитик всё чаще выступает не только как автор требований, но и как **дизайнер обмена** между системами.

### Зачем аналитику знать про iPaaS / ESB

Потому что именно здесь решаются вопросы:

- какие системы должны обмениваться данными;
- кто является master-источником;
- как часто и в каком формате идёт обмен;
- что делать при ошибке доставки;
- где нужны события, а где синхронный API;
- как обеспечить идемпотентность и трассируемость.

### Ключевые платформы интеграции

| Платформа | Особенности | Где сильна |
|-----------|------------|------------|
| **[MuleSoft Anypoint](https://www.mulesoft.com/platform/enterprise-integration)** | API-led connectivity | Enterprise и сложный API-ландшафт |
| **[Azure Integration Services](https://azure.microsoft.com/en-us/products/category/integration)** | Logic Apps, API Management, события и интеграции | Microsoft-стек, hybrid integration |
| **Boomi** | Быстрый старт и богатый каталог коннекторов | Средний и крупный бизнес |
| **SAP Integration Suite** | Интеграция с SAP-экосистемой | Enterprise и SAP-ландшафт |
| **Workato** | Business-friendly automation + integration | SaaS-heavy компании |
| **[DATAREON ESB](https://datareon.ru/)** | Российские интеграционные решения и шина данных | Корпоративные интеграции, 1С-контур |
| **[1С:Шина](https://v8.1c.ru/static/1c-shina/)** | ESB для обмена данными между системами | 1С-экосистема и enterprise integration |

### ESB vs iPaaS: когда что

| Критерий | ESB / on-prem | iPaaS / cloud |
|----------|---------------|---------------|
| **Развёртывание** | Собственная инфраструктура | Облако провайдера |
| **Контроль данных** | Максимальный | Зависит от провайдера и контура |
| **Скорость внедрения** | Ниже | Выше |
| **Где особенно актуально** | Legacy, regulated, 1С, госсектор | SaaS-ландшафты, быстрые интеграции |
| **Типичный российский выбор** | Часто приоритетен | Используется там, где допустимо облако |

## 8. BI и Data: аналитика с AI-ассистентами

К 2030 году работа с данными всё заметнее смещается от «сделай отчёт» к «дай ответ на вопрос по данным». Это не отменяет модель, семантический слой и data governance, но радикально снижает порог входа в аналитику для бизнеса.

### Как меняется работа с данными

| Сейчас | 2028–2030 |
|--------|-----------|
| Аналитик пишет SQL вручную | Бизнес задаёт вопрос на естественном языке |
| Дашборды строятся под заказ | Часть визуализаций и объяснений предлагает AI |
| BI-команда — узкое горлышко | Расширяется self-service модель |
| Отчёты статичны | AI подсказывает аномалии, драйверы и отклонения |

{% hint style="info" %}
BI будущего — это не «дашборды вместо аналитиков». Это среда, в которой аналитик отвечает за метрики, определения, интерпретацию и доверие к данным.
{% endhint %}

## 9. Диаграммы + AI: быстрее моделировать, проще обсуждать

Текстовые диаграммы становятся стандартным промежуточным форматом между идеей, обсуждением и системой. Для аналитика это удобно: диаграммы можно хранить рядом с требованиями, быстро менять, обсуждать в pull request и генерировать из описания.

### Полезные инструменты

| Инструмент | Для чего подходит |
|-----------|-------------------|
| **[Mermaid](https://mermaid.live/)** | Быстрое создание flowchart, sequence diagram, ERD, user journey |
| **PlantUML** | UML, C4, sequence и архитектурные схемы |
| **[Eraser](https://www.eraser.io/)** | AI-генерация архитектурных и продуктовых диаграмм |
| **LLM + Mermaid / PlantUML** | Черновое построение схем по текстовому описанию |

## Гиперавтоматизация: как всё связано

Эти инструменты не существуют по отдельности. В зрелой организации они образуют единую экосистему:

```text
                    ┌─────────────────────┐
                    │   Process Mining     │ ← Показывает реальный процесс
                    │  (Celonis, PIX, PM)  │
                    └────────┬────────────┘
                             ↓
┌──────────────┐    ┌────────────────────┐    ┌──────────────────┐
│   AI / LLM   │ ←→ │ BPM / Orchestration │ ←→ │   iPaaS / ESB    │
│ (GPT, Claude,│    │ (Camunda, ELMA365,  │    │ (MuleSoft,       │
│ GigaChat,    │    │ Flowable, BPMSoft)  │    │ DATAREON, 1С)    │
│ YandexGPT)   │    └────────┬────────────┘    └──────────────────┘
└──────────────┘             ↓
                    ┌────────────────────┐
                    │   RPA + Low-code   │ ← Исполняет рутинные задачи
                    │ (UiPath, n8n, PIX, │
                    │  Primo, Power Auto)│
                    └────────┬────────────┘
                             ↓
                    ┌────────────────────┐
                    │      BI / Data     │ ← Измеряет результат
                    │ (PBI, Tableau,     │
                    │ DataLens, Visiology)│
                    └────────────────────┘
```

**Роль аналитика** — видеть эту экосистему целиком и понимать, где задачу должен решать человек, где агент, где маршрут процесса, где интеграция, а где сначала нужно измерить фактический AS-IS.

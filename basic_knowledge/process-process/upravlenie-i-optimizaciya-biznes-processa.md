# Управление и оптимизация бизнес-процесса

| Грейд | Роль |
|-------|------|
| 🟠 Senior | 🅱️ BA |

> **Оптимизация процесса** — систематическое улучшение бизнес-процесса путем анализа данных, выявления проблем и внедрения изменений, которые повышают эффективность, качество и снижают затраты.

## Разница между управлением и оптимизацией

| Управление | Оптимизация |
|-----------|-----------|
| Следить, что процесс выполняется | Улучшать как процесс выполняется |
| "Это работает" | "Как сделать лучше?" |
| Реактивное (отвечаем на проблемы) | Проактивное (находим и исправляем) |
| День за днем | Стратегическое планирование |

## Цикл оптимизации процесса

```
Plan (Планирование) → Do (Внедрение) → Check (Проверка) → Act (Действие)
    ↑                                                       ↓
    └───────────────────────────────────────────────────────
```

## Шаг 1: Анализ текущего состояния (As-Is Analysis)

### Сбор данных

**Где брать информацию:**

1. **Наблюдение** — сидеть рядом с пользователем и смотреть
2. **Интервью** — спросить у людей, как они работают
3. **Логи системы** — время выполнения, ошибки, узкие места
4. **Метрики** — процент ошибок, среднее время, затраты

### Анализ данных

| Метрика | Как измерять | Норма | Проблема |
|---------|--------------|-------|----------|
| **Цикл-тайм** | От начала до конца | < 2 часов | > 5 часов |
| **Время ожидания** | Сколько задача ждет в очереди | 10% | 40% |
| **Частота ошибок** | % неправильно выполненных | < 1% | > 5% |
| **Cost per unit** | Затраты на 1 выполнение | $10 | $50 |
| **Текучесть** | % людей, которые ушли | < 10% | > 30% |

### Выявление проблем

Спросите себя:

- ❓ Где теряется больше всего времени?
- ❓ Где допускаются ошибки?
- ❓ Какие шаги не добавляют ценность клиенту?
- ❓ Что раздражает сотрудников?
- ❓ Что дорого стоит?

## Шаг 2: Проектирование улучшений (To-Be Design)

### Методы оптимизации

#### 1. Lean (Бережливое производство)
**Идея:** Убрать всё, что не добавляет ценность клиенту (waste)

**Типы waste:**
- Переработка (rework) — исправление ошибок
- Ожидание (waiting) — задача ждет в очереди
- Движение (motion) — лишние действия
- Запас (inventory) — незаконченные задачи

**Пример:**
```
ДО: Клиент пишет письмо → HR читает → Вводит в систему → Менеджер проверяет
    (много времени на передачу информации)

ПОСЛЕ: Клиент заполняет форму в системе → Менеджер видит автоматически
    (никаких промежуточных действий)
```

#### 2. Six Sigma
**Идея:** Снизить вариативность и ошибки до минимума (3.4 ошибки на миллион операций)

**Инструменты:** статистический анализ, тесты гипотез, DMAIC

#### 3. Reengineering (Полная переразработка)
**Идея:** Не совершенствовать существующий процесс, а переделать его заново

**Когда применять:** Когда Lean/Six Sigma не помогают, или процесс критически неэффективен

#### 4. Automatization (Автоматизация)
**Идея:** Автоматизировать повторяющиеся, предсказуемые действия

**Пример:**
- ручное: Менеджер отправляет письмо каждому клиенту
- автоматизировано: Система отправляет письма по расписанию

## Шаг 3: Пилотное внедрение (Pilot Implementation)

**Не внедряйте сразу для всех!**

### План пилота

1. **Выбрать небольшую группу** (20-50 человек, 1-2 недели)
2. **Обучить пилот-группу** на новый процесс
3. **Собрать обратную связь** (что хорошо, что плохо)
4. **Внести корректировки**
5. **Постепенно расширять** (25% → 50% → 100%)

### KPI для пилота

- Оценка удовлетворенности (1-5 баллов)
- Время на обучение
- Текучесть (кто ушел из пилот-группы?)
- Соответствие новому процессу

## Шаг 4: Масштабирование и мониторинг (Scale & Monitor)

### Dashboard для отслеживания

Создайте дашборд с ключевыми метриками:

```
┌──────────────────────────────────┐
│ Процесс: Оформление заказа      │
├──────────────────────────────────┤
│ Цикл-тайм: 1.5ч (было 3ч)       │✓
│ Ошибки: 0.5% (было 3%)          │✓
│ Стоимость: $12 (было $25)        │✓
│ Удовлетворение: 4.7/5 (было 3.2)│✓
│                                  │
│ Проблемы: 2 / Идеи: 5           │
└──────────────────────────────────┘
```

### Регулярный review

- **Еженедельно:** проверить нет ли проблем
- **Ежемесячно:** анализ метрик, обсуждение
- **Ежеквартально:** планирование следующих улучшений

## Шаг 5: Непрерывное совершенствование (Continuous Improvement)

**PDCA цикл как образ жизни:**

```
Неделя 1: Внедрили улучшение A
Неделя 2: Проверили результаты → Собрали feedback
Неделя 3: Внедрили улучшение B (на основе feedback)
Неделя 4: Проверили результаты → Готовимся к улучшению C
```

## Инструменты для анализа и оптимизации

| Инструмент | Что измеряет | Применение |
|-----------|-------------|-----------|
| **Process Mining** | Реальное выполнение процесса | Выявление узких мест в логах системы |
| **Value Stream Mapping** | Ценность каждого шага | Lean-анализ |
| **Fishbone Diagram** | Причины проблем | Поиск корневой причины |
| **Pareto Analysis** | 80/20 правило | Найти 20% проблем, которые создают 80% задержек |

## Факторы успеха оптимизации

✓ **Данные, а не мнения** — решения на основе метрик
✓ **Участие пользователей** — они знают реальные проблемы
✓ **Постепенные изменения** — не революция, а эволюция
✓ **Отслеживание результатов** — видно ли улучшение?
✓ **Регулярное обновление** — оптимизация никогда не кончается

## Типичные ошибки

❌ Оптимизировать на основе "как кажется" вместо данных
❌ Внедрить большие изменения сразу (нужен пилот)
❌ Забыть обучить сотрудников (они не поймут зачем)
❌ Оптимизировать сломанный процесс (сначала зафиксить)
❌ Остановиться на одном улучшении (это непрерывный процесс)

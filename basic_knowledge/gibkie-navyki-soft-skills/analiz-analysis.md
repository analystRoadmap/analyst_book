# Анализ (Analysis)

| Уровень | Роль |
|---------|------|
| 🟢 Junior | 🅱️🅰️ BA+SA |

> "Анализ - это разделение целого на части. Синтез - соединение частей в целое." - Неизвестный автор

## Определение

**Анализ** - это процесс детального изучения, разбора и оценки информации, данных, ситуации или явления с целью выявления их структуры, свойств, причин, последствий и возможных вариантов развития. В контексте работы аналитика анализ - это фундаментальный навык, который позволяет разложить сложную проблему на управляемые компоненты.

## Практические методы анализа для аналитиков

### 1. SWOT-анализ
Метод оценки сильных и слабых сторон, возможностей и угроз. Применяется для анализа текущего состояния бизнеса перед началом проекта.

**Как применять:**
- Создайте матрицу 2x2
- В каждый квадрант поместите факторы: Strengths, Weaknesses, Opportunities, Threats
- Используйте эту информацию для принятия стратегических решений

### 2. PEST-анализ
Оценка политических, экономических, социальных и технологических факторов, влияющих на бизнес.

**Когда использовать:** При анализе внешней среды и рисков проекта.

### 3. Анализ потребностей клиентов
Выявление потребностей и требований конечных пользователей, анализ болевых точек.

**Техники:**
- Интервью с пользователями
- Анкетирование
- Наблюдение (observation)
- User stories разработка

### 4. Анализ данных
Обработка, интерпретация и анализ больших объемов информации для выявления закономерностей и тренов.

**Инструменты:**
- Excel, Google Sheets
- Python (pandas, numpy)
- Power BI, Tableau
- SQL для работы с БД

### 5. Анализ процессов (Process Mining)
Оптимизация бизнес-процессов путем выявления узких мест и неэффективностей.

**Методы документирования:**
- BPMN-диаграммы
- Flow diagrams
- As-is / To-be анализ

### 6. Дерево проблем (Problem Tree)
Систематическое выявление корневых причин проблемы через построение иерархической структуры.

**Процесс:**
1. Определите главную проблему (ствол)
2. Выявите причины (корни)
3. Определите последствия (ветви)

### 7. Методика "5 почему" (5 Whys)
Определение глубинных причин проблемы через последовательное задание вопроса "Почему?".

**Пример из практики:**
- Проблема: Пользователи не используют новый функционал
- Почему? Они не знают, что он существует
- Почему? Отсутствует документация
- Почему? Документация не была включена в план проекта
- Почему? Недостаточное взаимодействие с командой
- Почему? Нет четкого процесса согласования документации

## Развитие навыка анализа

### Основные компетенции
1. **Умение работать с данными** - обработка больших объемов информации, выявление связей между данными, поиск ошибок
2. **Умение интерпретировать информацию** - понимание контекста и значения, оценка достоверности
3. **Умение формулировать проблемы** - четкое определение и разбор проблем, определение приоритетов
4. **Применение инструментов анализа** - использование методов и ПО для аналитической работы
5. **Системный взгляд** - видение взаимосвязей между элементами системы

### Практические упражнения
- Регулярно применяйте методы анализа в реальной работе
- Ведите дневник проектов - анализируйте, что сработало, а что нет
- Создавайте диаграммы для визуализации проблем
- Проводите post-mortem анализ завершенных проектов

## Распространенные ошибки при анализе

1. **Поверхностный анализ** - остановка на первом уровне причин
2. **Подтверждение предвзятости** - поиск только информации, подтверждающей вашу гипотезу
3. **Переанализ** - анализ без действий и решений
4. **Игнорирование контекста** - анализ без учета окружающих факторов
5. **Отсутствие данных** - принятие решений без достаточной информации

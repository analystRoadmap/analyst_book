# UX/UI Design

| Грейд | Роль |
|-------|------|

> **UX/UI** — совокупность дисциплин, которые превращают требования и процессы в интуитивно понятные и привлекательные интерфейсы, обеспечивающие лучший пользовательский опыт.

## Почему аналитику нужно понимать UX/UI?

Аналитик не проектирует интерфейсы, но он определяет, что должно быть видно пользователю и как он будет с этим взаимодействовать. Понимание основ UX/UI позволяет:

- Описывать требования в терминах, понятных дизайнерам
- Предусматривать потребности пользователей на этапе анализа
- Валидировать прототипы и макеты
- Участвовать в тестировании и оценке юзабилити

## UI Design Components

**Layout** — организация элементов интерфейса на экране

Типы:
- **Linear Layout** — элементы в строку/столбец
- **Grid Layout** — элементы в сетке
- **Constraint Layout** — адаптивные макеты для разных устройств
- **Flexbox** — гибкое распределение пространства

Для аналитика: при описании экрана указывайте, какие элементы должны быть видны и их приоритет.

---

**Color Palette** — цветовая система приложения

| Тип | Назначение | Пример |
|-----|-----------|--------|
| Primary | Основные действия, CTA | Кнопка "Отправить" |
| Secondary | Вспомогательные элементы | Ссылки, иконки |
| Success | Положительные результаты | Подтверждение, успех |
| Error | Ошибки и предупреждения | Сообщения об ошибках |
| Neutral | Фон, текст, разделители | Основной текст |

---

**Typography** — система шрифтов

- **Заголовки** — крупные, привлекают внимание
- **Основной текст** — легко читаемый, размер 14-16px
- **Подписи** — информация второго уровня
- **Code** — моноширинный для технических данных

Совет: указывайте в требованиях, где нужен тип данных (число, дата, ссылка).

---

**Icons** — визуальные символы действий и статусов

Типы:
- **Functional** — вызывают действия (поиск, загрузка)
- **Status** — показывают состояние (загрузка, ошибка)
- **Decorative** — улучшают восприятие (лайк, сохранение)

---

**Spacing** — промежутки между элементами

| Уровень | Использование |
|---------|---------------|
| **Compact (4-8px)** | Плотное расположение элементов |
| **Normal (16px)** | Стандартные промежутки |
| **Comfortable (24-32px)** | Просторный, современный вид |

## UX Research & Strategy

**User Stories** — описание функции глазами пользователя

Формат: "Как [роль], я хочу [действие], чтобы [результат]"

Пример:
```
Как менеджер, я хочу видеть историю заказов клиента,
чтобы быстро решить проблему с доставкой.
```

Для аналитика: каждое требование должно быть обосновано потребностью пользователя.

---

**Personas** — типичные образы пользователей

| Атрибут | Менеджер | Клиент |
|---------|----------|--------|
| Опыт | Высокий | Средний |
| Частота использования | Ежедневно | Раз в месяц |
| Главная цель | Управление | Совершение покупки |
| Боли | Медленный интерфейс | Сложная регистрация |

---

**User Flows** — путь пользователя через систему

```
Вход → Каталог → Выбор товара → Корзина → Оплата → Подтверждение
```

Аналитик определяет шаги, дизайнер визуализирует.

---

**Usability Testing** — проверка интерфейса с реальными пользователями

Метрики:
- **Task Completion Rate** — успешно ли пользователь выполнил задачу?
- **Time on Task** — сколько времени потребовалось?
- **Error Rate** — сколько ошибок пользователь допустил?
- **System Usability Scale (SUS)** — общая оценка удобства

## Design System

**Design System** — единая библиотека компонентов и правил

Включает:
- Компоненты (Button, Input, Card, Modal)
- Токены дизайна (цвета, размеры, интервалы)
- Паттерны взаимодействия
- Документация и примеры использования

Преимущества:
- Консистентность интерфейса
- Ускорение разработки
- Проще масштабировать продукт

## Сотрудничество аналитика и дизайнера

| Этап | Аналитик | Дизайнер |
|------|----------|----------|
| **Анализ требований** | Изучает потребности | - |
| **Пользовательские сценарии** | Описывает пути | - |
| **Wireframes** | Уточняет детали | Создает скелет |
| **Hi-Fi макеты** | Валидирует | Оформляет дизайн |
| **Прототип** | Готовит тесты | Интерактивный макет |
| **Usability test** | Готовит вопросы | Наблюдает |
| **Итерации** | Собирает feedback | Улучшает дизайн |

## Типичные ошибки в сотрудничестве

- ❌ Аналитик не передает достаточно контекста дизайнеру
- ❌ Требования описаны "как сделать" вместо "что получить"
- ❌ Отсутствует валидация с реальными пользователями
- ❌ Игнорируются feedback пользователей при тестировании

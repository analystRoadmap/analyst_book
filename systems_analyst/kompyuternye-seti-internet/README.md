> Компьютерные сети - это инфраструктура, которая позволяет устройствам обмениваться данными и взаимодействовать между собой.

## Почему аналитик должен понимать сети?

Понимание компьютерных сетей критично для системного аналитика при:
- Интеграции приложения с внешними системами через API или другие сетевые протоколы
- Оценке производительности и задержек при работе с сетью
- Анализе безопасности коммуникаций и защиты данных
- Проектировании распределенных систем
- Диагностике проблем с доступностью и скоростью

## Основные концепции

**TCP/IP Model**: четырехслойная модель, описывающая как данные передаются в интернете (Network Access, Internet, Transport, Application).

**OSI Model**: семислойная концептуальная модель (Physical, Data Link, Network, Transport, Session, Presentation, Application).

**DNS**: система преобразования доменных имен (example.com) в IP-адреса (192.168.1.1).

**HTTP/HTTPS**: протоколы передачи веб-содержимого, HTTPS с шифрованием для безопасности.

**Browser**: программа, которая делает запросы к веб-серверам и отображает контент.

**Hosting**: услуга размещения веб-приложений на серверах, доступных через интернет.

```
┌──────────────────────────────────────────────────────┐
│  Пользователь в браузере                             │
├──────────────────────────────────────────────────────┤
│  Интернет провайдер (ISP)                            │
│  Маршрутизаторы, кабели, Wi-Fi                       │

│  Корневые DNS-серверы (преобразуют имена)            │

│  Веб-серверы хостинг-провайдеров (например AWS)      │
├──────────────────────────────────────────────────────┤
│  Базы данных, хранилища (облако)                     │
└──────────────────────────────────────────────────────┘
```

## Ключевые показатели сетевой производительности

**Bandwidth (Пропускная способность)**: объем данных, передаваемый в единицу времени (Mbps, Gbps).

**Latency (Задержка)**: время в миллисекундах, требуемое для отправки пакета данных с одного компьютера на другой. Меньше - лучше.

**Jitter**: колебания латенции. Нестабильная сеть имеет высокий jitter.

**Packet Loss (Потеря пакетов)**: процент пакетов, которые не достигли получателя. Обычно < 1% для надежной сети.

**Throughput**: реальная скорость передачи полезных данных (отличается от bandwidth из-за overhead).

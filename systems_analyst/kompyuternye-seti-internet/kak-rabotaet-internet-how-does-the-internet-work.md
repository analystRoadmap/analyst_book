# Как работает интернет (How the Internet Works)

| Грейд | Роль |
|-------|------|

> Интернет - это глобальная сеть компьютеров, связанных между собой и обменивающихся данными посредством стандартизированных протоколов.

## Основные принципы работы интернета

Интернет работает по принципу **коммутации пакетов**: данные разбиваются на небольшие пакеты, каждый из которых может проходить независимый маршрут до точки назначения. Это отличается от старых систем коммутации каналов (как телефонные линии), где занимается весь канал на всё время разговора.

Ключевые компоненты:
- **TCP/IP протоколы**: обеспечивают надежную доставку данных между устройствами
- **IP-адреса**: уникальные адреса каждого устройства в интернете (например, 8.8.8.8)
- **Маршрутизаторы**: направляют пакеты по оптимальному пути
- **DNS**: преобразует доменные имена (google.com) в IP-адреса
- **Серверы**: компьютеры, которые хранят данные и отвечают на запросы

## Путь запроса в интернете

### 1. Вы вводите URL в браузер

```
https://www.example.com/page
```

### 2. DNS-разрешение

Браузер отправляет запрос на DNS-сервер:
```
Браузер: "Какой IP-адрес у www.example.com?"
DNS: "IP-адрес: 93.184.216.34"
```

### 3. TCP соединение

Браузер подключается к серверу по IP-адресу:
```
Браузер → Router → ISP → Internet Backbone → Другой Router → Web Server
```

### 4. HTTP запрос

Браузер отправляет HTTP GET запрос:
```
GET /page HTTP/1.1
Host: www.example.com
User-Agent: Chrome/120.0
```

### 5. Web сервер обрабатывает

Сервер:
- Получает запрос
- Ищет файл /page
- Обрабатывает логику (если это динамический контент)
- Возвращает HTTP ответ:

```
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234

<html>
  <body>
    <h1>Добро пожаловать!</h1>
  </body>
</html>
```

### 6. Браузер получает и отображает

Браузер получает HTML, парсит его, загружает CSS и JavaScript, отображает страницу пользователю.

## Как пакеты проходят через интернет

Данные разбиваются на пакеты размером обычно 1500 байт. Каждый пакет содержит:
- **IP заголовок**: от кого и кому
- **TCP заголовок**: порт назначения, номер пакета
- **Данные**: полезная нагрузка

Пакеты проходят через множество маршрутизаторов, каждый выбирает оптимальный путь на основе таблиц маршрутизации.

Пример маршрута пакета:
```
Ваш компьютер → Ваш Router → ISP Router → Интернет Backbone →
Другой ISP → Другой Router → Web Server
```

## Скорость и задержки

**Скорость света**: данные передаются со скоростью света через волокна оптических кабелей (~200,000 км/сек).

**Геолокация**: при отправке пакета из Москвы в Нью-Йорк расстояние ~10,000 км, минимальное время передачи ~50 мс, но реально около 80-100 мс из-за маршрутизации.

**Процесс передачи большого файла**:
- Файл 10 МБ (10,000,000 байт)
- Разбивается на ~6,666 пакетов
- Каждый пакет проходит независимый маршрут
- Браузер ждет, пока все пакеты придут, в правильном порядке
- TCP гарантирует надежность доставки

## Уровни интернета

### Физический уровень
- Кабели (медь, волокно)
- Wi-Fi и другие радиоволны
- Спутниковый интернет

### Сетевой уровень (IP)
- Маршрутизация
- IP-адреса (IPv4 и IPv6)
- Определение пути доставки

### Транспортный уровень (TCP/UDP)
- TCP: гарантированная доставка с проверкой ошибок
- UDP: быстрая доставка без гарантий

### Прикладной уровень
- HTTP/HTTPS для веб
- SMTP для email
- SSH для удаленного доступа

## Пример для аналитика

При анализе проблемы "веб-сайт медленно загружается", нужно рассмотреть:

1. **Клиентская сторона**: сколько времени браузер затрачивает на DNS, TCP соединение, получение и отображение
2. **Сетевая задержка**: пинг до сервера (команда `ping example.com`)
3. **Пропускная способность**: скорость интернета пользователя (Mbps)
4. **Сервер**: как быстро сервер обрабатывает запрос

Типичный breakdown для HTTP запроса:
- DNS: 20-50 ms
- TCP соединение: 50-100 ms
- HTTP запрос и обработка: 50-200 ms
- Передача ответа: зависит от размера
- Отображение в браузере: 100-500 ms

## Безопасность в интернете

**HTTP vs HTTPS**: HTTPS шифрует данные при передаче (SSL/TLS протокол), HTTP нет. HTTPS обязателен для любых чувствительных данных.

**Man-in-the-Middle атака**: без HTTPS, злоумышленник может перехватить пакеты и увидеть пароли.

**Certificates**: HTTPS требует сертификата, который доказывает, что вы подключены к правильному серверу, а не подделке.

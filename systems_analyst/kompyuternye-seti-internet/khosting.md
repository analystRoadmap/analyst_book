# Хостинг (Web Hosting)

> Хостинг - это услуга размещения веб-приложений и сайтов на серверах, подключенных к интернету, чтобы они были доступны пользователям 24/7.

## Что такое хостинг?

Хостинг-провайдеры предоставляют инфраструктуру (физические или виртуальные серверы) для размещения веб-сайтов и приложений. Вместо того, чтобы выделять и управлять собственным сервером, компания платит провайдеру за использование их ресурсов.

Основные компоненты хостинг-провайдера:
- Физические серверы в дата-центрах
- Сетевые соединения с гарантированной пропускной способностью
- Техническая поддержка
- Услуги резервного копирования и восстановления
- Система мониторинга и алертинга

## Основные компоненты хостинга

**Серверы**: физические или виртуальные компьютеры, которые хранят файлы вашего приложения и обрабатывают запросы пользователей. Серверы работают 24/7.

**Хранилище (Storage)**: дисковое пространство для файлов, БД, медиа-ресурсов. Обычно измеряется в GB или TB.

**Пропускная способность (Bandwidth)**: объем данных, который можно передать пользователям в месяц. При превышении либо взимается дополнительная плата, либо сайт временно блокируется.

**Доменное имя**: адрес сайта (example.com). Регистрация может быть отдельно или включена в хостинг.

**Email хостинг**: возможность создавать email ящики вида user@yourdomain.com.

**Техническая поддержка**: помощь в установке, настройке, устранении проблем.

## Типы хостинга

### 1. Shared Hosting (Общий хостинг)

Несколько сайтов размещаются на одном физическом сервере и делят его ресурсы.

**Плюсы**:
- Самый дешевый вариант ($5-15/месяц)
- Просто в управлении, особенно для новичков
- Техподдержка помогает в администрировании

**Минусы**:
- Ограниченные ресурсы (CPU, RAM)
- Если один сайт перегрузится, другие пострадают
- Нет полного контроля над окружением
- Медленнее, чем другие варианты

**Когда использовать**:
- Личные блоги
- Маленькие коммерческие сайты
- Сайты с низкой нагрузкой (< 100 посетителей в день)

### 2. VPS (Virtual Private Server)

Один физический сервер разделен на несколько виртуальных серверов, каждый имеет выделенные ресурсы.

**Плюсы**:
- Гарантированные ресурсы (не зависят от других пользователей)
- Root доступ, полный контроль над окружением
- Можно установить любое ПО
- Лучшая производительность

**Минусы**:
- Дороже shared hosting ($15-50/месяц)
- Требуется знание администрирования Linux/Windows
- Самостоятельное управление обновлениями и безопасностью

**Когда использовать**:
- Растущие веб-приложения
- Сайты с переменной нагрузкой
- Когда нужен полный контроль

### 3. Dedicated Server (Выделенный сервер)

Весь физический сервер принадлежит вам.

**Плюсы**:
- Максимальная производительность
- Полный контроль
- Лучшая для высоконагруженных систем

**Минусы**:
- Самый дорогой вариант ($100-500+/месяц)
- Требуется опыт администрирования
- Фиксированные ресурсы (сложно масштабировать вверх)

**Когда использовать**:
- Высоконагруженные сайты (> 1 млн посетителей/месяц)
- Специфические требования к оборудованию
- Высокие требования к безопасности

### 4. Cloud Hosting (Облачный хостинг)

Приложение размещается на виртуальных серверах в облаке (AWS, Google Cloud, Azure).

**Плюсы**:
- Масштабируемость (добавить ресурсы за минуты)
- Платишь только за использованное
- Высокая доступность (сервер упал - автоматический переход на другой)
- Pay-as-you-go модель

**Минусы**:
- Сложнее в управлении
- Переменные затраты (сложно предсказать счет)
- Требуется знание облачной инфраструктуры

**Когда использовать**:
- Масштабируемые приложения
- Переменная нагрузка (пики трафика)
- Требуется высокая доступность
- Приложения, которые быстро растут

### 5. Serverless

Функции размещаются на управляемых платформах (AWS Lambda, Google Cloud Functions).

**Плюсы**:
- Платишь только за использованное
- Автоматическое масштабирование
- Полностью управляемая инфраструктура
- Хорошо для событийно-ориентированных приложений

**Минусы**:
- Ограничения на время выполнения
- Может быть дорого при высокой нагрузке
- Привязка к провайдеру

**Когда использовать**:
- API и микросервисы
- Фоновые задачи
- Обработка событий

## SLA (Service Level Agreement)

Хостинг-провайдеры гарантируют **availability** - процент времени, когда сервис работает:

- **99.5%** (четыре девятки): 3.6 часа downtime в год
- **99.9%** (три девятки): 8.76 часов downtime в год
- **99.99%** (четыре девятки): 52.5 минут downtime в год
- **99.999%** (пять девяток): 5.25 минут downtime в год

Больше девяток = более дорогой сервис.

## CDN (Content Delivery Network)

CDN - это сеть серверов по всему миру для быстрой доставки контента.

Вместо того, чтобы все пользователи скачивали из одного сервера:
```
Пользователь в Москве → Сервер в Нью-Йорке (медленно, ~100ms)
```

CDN доставляет с ближайшего сервера:
```
Пользователь в Москве → Сервер в Москве (быстро, ~10ms)
```

Популярные CDN:
- Cloudflare (DDoS защита, кэширование)
- Akamai (большая сеть)
- Amazon CloudFront (интеграция с AWS)
- Yandex CDN (для России)

## Backup и Disaster Recovery

Хостинг-провайдеры должны обеспечить:
- **Daily Backup**: ежедневное резервное копирование
- **Off-site Storage**: хранение копий в другом месте
- **Recovery Time Objective (RTO)**: как быстро восстановить при сбое
- **Recovery Point Objective (RPO)**: максимум потеряемых данных

Для критичных приложений требуется проверенный процесс восстановления.

## Безопасность хостинга

**SSL/HTTPS**: шифрование трафика

**Firewall**: защита от несанкционированного доступа

**DDoS Protection**: защита от атак

**Regular Updates**: патчи безопасности

**Access Control**: кто может получить доступ к серверу

## Практический пример для аналитика

### Выбор хостинга для стартапа

Фаза 1 (0-1000 пользователей в месяц):
- Shared Hosting ($10/месяц)
- Достаточно для MVP
- Просто управлять

Фаза 2 (1000-100,000 пользователей):
- VPS ($30/месяц)
- Гарантированные ресурсы
- Лучше контроль

Фаза 3 (>100,000 пользователей):
- Cloud Hosting (AWS, Google Cloud)
- Масштабируемость
- Платишь за использованное

### Оценка затрат

Shared Hosting:
- $10/месяц × 12 = $120/год
- Если перегрузится, нужно переходить

Cloud Hosting для постоянной нагрузки 100 RPS:
- EC2 (small): $30/месяц
- Database: $20/месяц
- CDN: $10-50/месяц
- Итого: $60-100/месяц × 12 = $720-1200/год

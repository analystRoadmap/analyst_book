> HTTP (Hyper Text Transfer Protocol) и HTTPS - это протоколы передачи данных в интернете, на которых основан весь веб.

## Что такое HTTP?

HTTP (HyperText Transfer Protocol) - это протокол на уровне приложения (Layer 7), который определяет, как клиенты (браузеры) и серверы обмениваются данными в интернете.

Основной принцип: **запрос-ответ**
1. Клиент отправляет HTTP запрос на сервер
2. Сервер обрабатывает и отправляет HTTP ответ
3. Соединение может быть закрыто или переиспользовано

## Как работает HTTP

### HTTP Запрос

```
GET /page HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html
Connection: keep-alive
```

Структура запроса:
- **Method**: GET, POST, PUT, DELETE, PATCH и другие
- **Path**: /page - какой ресурс запрашивается
- **HTTP Version**: 1.0, 1.1, 2, 3
- **Headers**: дополнительная информация (User-Agent, Accept)
- **Body**: данные (в POST запросах)

### Методы HTTP

| Метод | Описание | Пример |
|-------|---------|--------|
| **GET** | Получить данные | Получить страницу, список товаров |
| **POST** | Создать новый ресурс | Отправить форму, создать заказ |
| **PUT** | Полное обновление ресурса | Обновить профиль пользователя целиком |
| **PATCH** | Частичное обновление | Изменить только имя пользователя |
| **DELETE** | Удалить ресурс | Удалить товар из корзины |
| **HEAD** | GET но без тела ответа | Проверить существование ресурса |
| **OPTIONS** | Узнать доступные методы | Какие методы поддерживает API |

### HTTP Ответ

```
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234
Set-Cookie: sessionid=abc123

<html>
  <body>
    <h1>Добро пожаловать!</h1>
  </body>
</html>
```

Структура ответа:
- **Status Code**: 200, 404, 500 и другие
- **Headers**: информация об ответе
- **Body**: содержимое (HTML, JSON, видео и т.д.)

## HTTP Status Codes

| Код | Категория | Примеры | Значение |
|-----|-----------|---------|----------|
| **2xx** | Success | 200 OK, 201 Created | Успешно обработано |
| **3xx** | Redirection | 301 Moved Permanently, 302 Found | Редирект на другой URL |
| **4xx** | Client Error | 400 Bad Request, 404 Not Found, 403 Forbidden | Ошибка клиента |
| **5xx** | Server Error | 500 Internal Server Error, 503 Service Unavailable | Ошибка сервера |

## HTTP vs HTTPS

**HTTP**: передача данных в открытом виде, небезопасно
- Злоумышленник может перехватить трафик (Man-in-the-Middle атака)
- Может увидеть пароли, личные данные

**HTTPS**: шифрованная передача данных через SSL/TLS протокол
- Данные зашифрованы, только получатель может прочитать
- Требует сертификат, который доказывает подлинность сервера
- Стандарт безопасности современного веб

### Как работает HTTPS

1. **Handshake**: браузер и сервер договариваются о способе шифрования
2. **Certificate Exchange**: сервер отправляет сертификат, подтверждающий его подлинность
3. **Key Exchange**: договариваются о шифровании (часто используется алгоритм TLS)
4. **Encrypted Communication**: все данные передаются в зашифрованном виде

Пользователь видит закрытый замок в адресной строке браузера, когда соединение безопасно.

## HTTP Version эволюция

**HTTP/1.0** (1996):
- Каждый запрос требует отдельного TCP соединения
- Медленно для множества ресурсов

**HTTP/1.1** (1997):
- Keep-Alive: переиспользование TCP соединения
- Множество улучшений производительности

**HTTP/2** (2015):
- Multiplexing: несколько запросов в одном соединении
- Server Push: сервер может отправить данные без запроса
- Лучше для медленных сетей

**HTTP/3** (2022):
- Основан на UDP вместо TCP (быстрее)
- Еще лучшая производительность
- Еще не везде распространен

## REST API и HTTP

REST (Representational State Transfer) - архитектурный стиль для API, использующий HTTP методы и статус коды.

Примеры REST API:

```
GET /api/users           → получить список пользователей (200 OK)
GET /api/users/5         → получить пользователя 5 (200 OK)
POST /api/users          → создать пользователя (201 Created)
PUT /api/users/5         → обновить пользователя 5 (200 OK)
DELETE /api/users/5      → удалить пользователя 5 (204 No Content)
GET /api/users/999       → пользователь не существует (404 Not Found)
```

## HTTP Headers

Важные headers для аналитика:

| Header | Назначение | Пример |
|--------|-----------|--------|
| **Content-Type** | Тип содержимого | application/json, text/html |
| **Content-Length** | Размер тела ответа | 1234 |
| **Authorization** | Авторизация | Bearer token123abc |
| **Cache-Control** | Кэширование | max-age=3600 (кэш на 1 час) |
| **User-Agent** | Информация о клиенте | Mozilla/5.0 (Chrome) |
| **Set-Cookie** | Установка cookie | sessionid=abc123; Path=/ |
| **X-Rate-Limit** | Rate limiting | X-Rate-Limit-Remaining: 99 |

## Безопасность

**HTTP без HTTPS**:
- Данные видны в открытом виде
- Может быть атака Man-in-the-Middle
- Никогда не используйте для логинов, платежей, личных данных

**HTTPS обязателен для**:
- Форм логина (пароли)
- Платежных систем (номера карт)
- Любых личных данных
- API, требующих аутентификацию

**Сертификаты**:
- Выданы Certification Authorities (CA)
- Имеют срок действия (например, 1 год)
- Должны быть продлены, иначе браузер выдаст ошибку

## Практический пример для аналитика

Анализ медленного загрузки сайта:

1. **Количество запросов**: сколько HTTP запросов отправляет браузер?
   - Запрос основной страницы (1)
   - Запросы CSS файлов (3-5)
   - Запросы JavaScript (3-5)
   - Запросы изображений (10+)
   - Запросы к API (5-10)
   - Итого: 30-50 запросов

2. **Размер данных**: сколько КБ/МБ загружается?
   - HTML: 50 KB
   - CSS: 100 KB
   - JavaScript: 200 KB
   - Изображения: 1-2 MB
   - Итого: 1.5-2.5 MB

3. **Способы оптимизации**:
   - Минификация CSS/JS (уменьшить размер)
   - Сжатие изображений (JPEG вместо PNG)
   - Lazy loading (загружать изображения при скролле)
   - CDN (доставлять данные с сервера ближе к пользователю)
   - Кэширование (браузер не перезагружает неизменные файлы)

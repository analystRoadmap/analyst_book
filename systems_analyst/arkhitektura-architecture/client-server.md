# Клиент-серверная архитектура (Client-Server Architecture)

| Грейд | Роль |
|-------|------|
| 🔵 Middle | 🅰️ Systems Analyst |

> Клиент-серверная архитектура - базовая модель распределенной системы, где клиент инициирует запросы, а сервер обрабатывает их и возвращает результаты.

## Что такое клиент-серверная архитектура?

Клиент-серверная архитектура разделяет систему на две части:
- **Клиент**: инициирует запросы, представляет интерфейс пользователя
- **Сервер**: обрабатывает запросы, управляет данными, предоставляет сервисы

Примеры:
- Веб-приложения (браузер как клиент, веб-сервер)
- Email (приложение для чтения писем как клиент, mail сервер)
- Корпоративные приложения (толстый клиент или браузер, приложение сервер)
- Игры онлайн (игровой клиент, игровой сервер)

## Основные компоненты

**Клиент**: программа на компьютере или мобильном устройстве пользователя. Отправляет запросы и отображает результаты. Может быть веб-браузером, десктопным приложением или мобильным приложением.

**Сервер**: программа на удаленном компьютере. Слушает входящие соединения, обрабатывает запросы, обращается к БД, возвращает результаты.

**Коммуникационная сеть**: связывает клиента и сервер. Используются протоколы TCP/IP, HTTP, HTTPS для безопасной передачи данных.

## Процесс взаимодействия

1. Клиент подключается к серверу
2. Клиент отправляет запрос (например, HTTP GET)
3. Сервер получает и обрабатывает запрос
4. Сервер обращается к БД или другим ресурсам
5. Сервер отправляет результат (HTTP 200 OK с данными)
6. Клиент получает ответ и отображает результат пользователю

## Основные преимущества

| Преимущество | Описание |
|---|---|
| **Централизованное управление** | Сервер управляет всеми данными и бизнес-логикой в одном месте |
| **Безопасность** | Данные хранятся и защищаются на сервере, клиент не может напрямую изменить данные |
| **Масштабируемость** | Можно добавить несколько серверов для распределения нагрузки, клиент ничего не знает об этом |
| **Единая версия** | Только один сервер, поэтому всегда одна версия данных и логики |
| **Универсальность** | Клиенты могут быть разных типов (веб, мобиль, десктоп) и взаимодействовать с одним сервером |

## Ограничения и вызовы

**Единая точка отказа**: если сервер упадет, все клиенты потеряют доступ к сервису. Решение - резервные серверы, балансировка нагрузки.

**Сетевая задержка**: каждый запрос требует сетевой коммуникации, что добавляет задержку. Нельзя делать 1000 запросов в секунду от одного клиента.

**Нагрузка на сервер**: при большом количестве клиентов сервер может перегрузиться. Требуется горизонтальное масштабирование.

**Зависимость от сети**: при плохом интернет соединении клиент не может работать.

## Типы клиент-серверных архитектур

**Two-Tier (Двухуровневая)**:
- Клиент напрямую подключается к БД
- Используется в локальных сетях
- Пример: толстый клиент с прямым подключением к SQL Server

**Three-Tier (Трехуровневая)**:
- Клиент → Application Server → Database
- Клиент не знает про БД напрямую
- Используется в веб-приложениях
- Пример: веб-браузер → Node.js сервер → PostgreSQL

**N-Tier (Многоуровневая)**:
- Добавляются промежуточные сервисы, API Gateway, кэши
- Клиент → API Gateway → Microservices → Database/Cache

## Пример для аналитика

Интернет-магазин на базе клиент-серверной архитектуры:

```
[Браузер пользователя]
         ↓ HTTP/HTTPS
[Web Server]  ←  получает запрос
         ↓
[Application Server]  ←  обработка логики добавления товара в корзину
         ↓
[Database Server]  ←  сохраняет корзину в БД
         ↓
[Response]  ←  браузер обновляет интерфейс корзины
```

При масштабировании можно:
- Добавить несколько веб-серверов за Load Balancer
- Кэшировать часто запрашиваемые данные (Redis)
- Использовать CDN для доставки статических файлов
- Реплицировать БД для высокой доступности

## Сравнение с другими архитектурами

| Архитектура | Клиент-Сервер | Микросервисы | P2P |
|---|---|---|---|
| Сложность | Низкая | Высокая | Средняя |
| Масштабируемость | Хорошая | Отличная | Отличная |
| Централизованное управление | Да | Нет | Нет |
| Задержка сети | Средняя | Высокая | Низкая |

# Serverless (Бессерверные вычисления)

| Грейд | Роль |
|-------|------|
| 🟠 Senior | 🅰️ Systems Analyst |

> Serverless - это парадигма, где разработчик платит только за использованные ресурсы, а управление инфраструктурой берет на себя облачный провайдер.

## Что такое Serverless?

Serverless (бессерверные вычисления) - архитектурный подход, при котором разработчики развертывают функции кода без необходимости управлять серверами, виртуальными машинами или контейнерами. Все задачи по масштабированию, мониторингу и выделению ресурсов берет на себя облачный провайдер.

Примеры Serverless платформ:
- AWS Lambda
- Google Cloud Functions
- Azure Functions
- Yandex Cloud Functions

## Основные характеристики

**Событийно-ориентированная архитектура**: код выполняется в ответ на события (HTTP-запросы, изменения в БД, загрузка файлов, сообщения в очередях). Функция срабатывает только когда это необходимо.

**Плата на основе использования**: оплачивается количество выполнений функции и время их работы (обычно в миллисекундах). Нет затрат на простои и неиспользуемые ресурсы.

**Автоматическое масштабирование**: платформа автоматически выделяет ресурсы в зависимости от количества запросов, без необходимости ручного управления.

**Ограниченные ресурсы**: функции выполняются в контролируемой среде с лимитами памяти (обычно 128MB-10GB) и времени выполнения (15 минут для Lambda).

## Преимущества Serverless

| Преимущество | Описание |
|---|---|
| **Экономия затрат** | Оплата только за использованное, идеально для переменных нагрузок |
| **Быстрое развертывание** | Функции развертываются мгновенно, каждая независимо |
| **Масштабируемость** | Автоматическое масштабирование при пиковых нагрузках |
| **Простота разработки** | Разработчик фокусируется на коде, а не на инфраструктуре |
| **Высокая доступность** | Встроенная отказоустойчивость и распределение нагрузки |
| **Интеграция с облачными сервисами** | Простая интеграция с БД, очередями, хранилищами |

## Ограничения и вызовы

**Cold Start**: при первом запуске или после простоя происходит задержка (от сотен миллисекунд до нескольких секунд) для инициализации окружения. Критично для приложений с требованиями к мгновенной реакции.

**Ограничение времени выполнения**: функция должна завершиться за отведенное время (обычно 15 минут), невозможно запустить долгие процессы.

**Контроль над инфраструктурой**: ограниченная возможность влиять на настройки сервера, мониторинг и оптимизацию под специфические требования.

**Сложность отладки**: локальная разработка требует эмуляции окружения, распределенные логи и трассировка затрудняют диагностику проблем.

**Зависимость от провайдера**: привязка к определенной платформе, миграция между провайдерами требует переписания функций.

## Когда использовать Serverless?

Serverless хорошо подходит для:
- REST API с переменной нагрузкой
- Обработка событий (загрузка файлов, изменения в БД)
- Фоновые задачи и планировщики
- Обработка вебхуков от внешних систем
- Микросервисы с низкой частотой вызовов

Избегайте Serverless для:
- Приложений, требующих постоянной готовности к низким латенциям
- Процессов выполняющихся дольше нескольких минут
- Систем с высокими вычислительными требованиями
- Приложений, сильно зависимых от состояния

## Архитектурные паттерны

**Function as a Service (FaaS)**: каждая функция - отдельный развертываемый компонент с собственным вычислением.

**Event-driven**: функции срабатывают на события от других сервисов, типовой паттерн для Serverless архитектур.

**API Gateway + Lambda**: HTTP запросы через API Gateway маршрутизируются на соответствующие Lambda функции.

## Пример для аналитика

При проектировании системы обработки загрузок документов:

Вместо выделенного сервера с постоянно работающим процессом (дорого при низкой активности), используем:
1. API Gateway принимает загрузку документа
2. Lambda функция срабатывает, преобразует документ
3. Результат сохраняется в S3/БД
4. Функция завершается, ресурсы освобождаются

При 100 загрузках/день платите за 100 выполнений, при 100,000 загрузках/день система автоматически масштабируется.

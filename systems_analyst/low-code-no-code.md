# Low-code и No-code платформы

> Low-code и no-code платформы позволяют быстро создавать приложения без написания кода. Аналитику важно знать возможности и ограничения для правильного выбора подхода.

## Различие между Low-code и No-code

### Low-code платформы

**Low-code** — визуальное создание приложений с возможностью написания кода для расширений.

**Особенности:**
- Визуальный конструктор для основной функциональности
- Возможность написания кода для сложных сценариев
- Требует технических знаний (не обязательно deep)
- Примеры: Mendix, OutSystems, Microsoft Power Apps

**Подходит для:**
- Корпоративные приложения
- Когда нужна гибкость
- Когда есть разработчики, но хочется ускорить

### No-code платформы

**No-code** — полностью визуальное создание без написания кода.

**Особенности:**
- Конструктор drag-and-drop
- Не требует технических знаний
- Ограничения в кастомизации
- Примеры: Bubble, Retool, n8n, Make

**Подходит для:**
- Простые приложения и автоматизация
- Бизнес-пользователи без технического образования
- MVP и прототипы

## Сравнение подходов

| Критерий | Low-code | No-code |
|----------|----------|---------|
| **Скорость разработки** | 2-3x быстрее чем код | 5-10x быстрее чем код |
| **Гибкость** | Высокая (можно писать код) | Средняя (только визуальное) |
| **Сложность требований** | До очень сложных | До средних |
| **Требуемые навыки** | Программист + низкокодовые знания | Бизнес-пользователь |
| **Масштабируемость** | Хорошая | Ограниченная |
| **Стоимость** | Средняя | Низкая |

## Популярные платформы

### Для низкокодовой разработки

**Mendix (Siemens)**
- Полноценная разработка мобильных и веб-приложений
- Мощная модель данных
- Интеграция с системами
- Подходит для: крупных корпоративных систем

**Microsoft Power Platform**
- Power Apps: создание приложений
- Power Automate: автоматизация процессов
- Интеграция с экосистемой Microsoft
- Подходит для: организаций, использующих Microsoft 365

**OutSystems**
- Быстрая разработка мобильных и веб-приложений
- Встроенный AI для генерации UI
- Ориентирована на производительность
- Подходит для: быстрое проектирование и MVP

### Для бескодовой разработки

**Bubble**
- Полностью визуальный конструктор
- Встроенная БД
- Хостинг в облаке
- Подходит для: стартапы, MVP, приложения для малого бизнеса

**Retool**
- Специализирована на админ-панелях и дашбордах
- Быстрое подключение к БД
- Поддержка множества источников данных
- Подходит для: внутренние инструменты, admin dashboards

**n8n**
- Автоматизация рабочих процессов
- Подключение к 300+ интеграциям
- Поддержка self-hosting
- Подходит для: интеграция систем, workflow automation

## Когда использовать Low-code/No-code

### Когда ЭТО подходит

✅ CRUD операции (создание, чтение, обновление, удаление)
✅ Простые бизнес-процессы
✅ Внутренние инструменты и дашборды
✅ Быстрые MVP и прототипы
✅ Интеграция существующих систем
✅ Автоматизация повторяющихся задач

**Пример:** CRM система для малой компании — 80% можно сделать в no-code.

### Когда это НЕ подходит

❌ Высоконагруженные системы (миллионы транзакций/сек)
❌ Сложные алгоритмы (ML, обработка изображений)
❌ Специфичные интеграции со старыми системами
❌ Критичные системы, требующие полного контроля
❌ Требования к глубокой кастомизации

**Пример:** платежный шлюз требует низкокодовой разработки + доп. разработчиков для интеграций.

## Процесс разработки на Low-code

### Шаг 1: Требования

Аналитик определяет:
- Сущности (User, Product, Order)
- Атрибуты каждой сущности
- Связи между сущностями
- Бизнес-процессы

### Шаг 2: Моделирование данных

На платформе создается модель данных:

```
Entity: Product
- id (UUID)
- name (String, 255)
- description (Text)
- price (Decimal)
- category_id (FK → Category)
- created_at (DateTime)
- updated_at (DateTime)
```

### Шаг 3: Разработка UI

Визуальный конструктор для интерфейса:

```
Page: Products List
- Header: "Products"
- Search bar
- Filter by category
- List of products (grid)
  - Each item:
    - Image
    - Name
    - Price
    - "Details" button → redirect to product detail page
```

### Шаг 4: Реализация логики

Связывание интерфейса с данными и логикой:

```
Button "Add to Cart":
1. Get product_id from current product
2. Call API: POST /api/cart/items
   payload: { product_id, quantity: 1 }
3. If success → show "Added to cart!"
4. If error → show error message
```

### Шаг 5: Тестирование и развертывание

- Тестирование на тестовой среде
- Развертывание в production
- Мониторинг ошибок

## Примеры использования Low-code/No-code

### Пример 1: CRM для малого бизнеса

**Требование:** Система управления контактами и сделками.

**Решение на Power Apps:**
1. Model-driven app с таблицами: Contact, Deal, Activity
2. Автоматизация процессов через Power Automate:
   - При создании сделки → отправить email менеджеру
   - При завершении сделки → обновить финансовые отчеты
3. Интеграция с Outlook для отправки email

**Преимущества:**
- Разработка за 2-3 недели вместо 3 месяцев
- Дешевле (лицензии низкокодовых платформ дешевле разработчиков)
- Просто расширять (добавить новое поле просто)

### Пример 2: Автоматизация интеграции систем

**Требование:** Синхронизировать контакты из CRM в маркетинг-платформу.

**Решение на n8n:**
1. Триггер: новый контакт создан в CRM
2. Трансформация: привести данные к формату маркетинг-платформы
3. Action: загрузить контакт в маркетинг-платформу

**Код workflow:**
```
Trigger: CRM Webhook (new contact created)
  ↓
Transform: Extract email, name, phone
  ↓
Action: POST to Marketing Platform API
  ↓
On Error: Send Slack notification
```

**Преимущества:**
- Разработка за 1 час вместо 1 дня
- Легко менять логику
- Мониторинг через UI

## Риски и ограничения

### Vendor Lock-in

Приложение сильно зависит от платформы. Миграция на другую платформу = переписание всего.

**Как минимизировать:**
- Выбирать платформы с экспортом кода
- Использовать стандартные API
- Отделить критичную логику от платформы

### Производительность

Низкокодовые приложения могут быть медленнее нативных из-за:
- Слоя абстракции
- Неоптимального сгенерированного кода
- Ограничений в кэшировании

### Безопасность

Платформа может генерировать код с уязвимостями. Нужен security аудит.

## Рекомендации для аналитика

### При выборе платформы

1. **Определить требования** — может ли платформа реализовать 80% функционала?
2. **Проверить интеграции** — поддерживает ли нужные системы?
3. **Оценить стоимость** — разработка + хостинг + лицензии
4. **Найти специалистов** — есть ли на рынке разработчики?
5. **Тест на MVP** — сделать прототип перед полной разработкой

### При работе с требованиями

- **Адаптируйте процессы** — иногда лучше изменить бизнес-процесс под платформу
- **Документируйте ограничения** — четко описывайте, что невозможно
- **Планируйте расширения** — как будут добавляться новые функции потом?
- **Будьте реалистичны** — не все требования реализуются на выбранной платформе

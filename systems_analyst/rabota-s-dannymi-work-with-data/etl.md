# ETL

| Грейд | Роль |
|-------|------|
| 🔵 Middle | 🅰️ System Analyst |

> ETL — фундаментальный процесс, через который данные попадают из источников в хранилище. Аналитик не пишет ETL-пайплайны, но проектирует требования к ним: что извлекать, как трансформировать, куда загружать и с какой периодичностью.

## Что такое ETL

ETL — Extract, Transform, Load — три последовательных шага обработки данных:

### Extract (Извлечение)
Получение данных из источников: базы данных, API, файлы (CSV, Excel), логи, внешние сервисы.

**Пример:** каждую ночь загружаем список новых заказов из OLTP-базы интернет-магазина.

### Transform (Трансформация)
Очистка и подготовка данных: приведение форматов, удаление дубликатов, обогащение, агрегация, валидация.

**Пример:** привести даты к единому формату (ISO 8601), убрать тестовые заказы, рассчитать поле `total_with_discount`.

### Load (Загрузка)
Запись подготовленных данных в целевую систему: Data Warehouse, Data Lake, BI-инструмент.

**Пример:** загрузить обработанные данные в ClickHouse для аналитических дашбордов.

## ETL vs ELT: в чём разница

| | ETL | ELT |
|---|-----|-----|
| **Порядок** | Extract → Transform → Load | Extract → Load → Transform |
| **Где трансформация** | На промежуточном сервере | В целевой БД |
| **Когда использовать** | Данных немного, нужна предобработка | Мощная целевая БД (BigQuery, Snowflake) |
| **Тренд** | Классический подход | Современный подход (cloud-first) |

Современные облачные DWH (BigQuery, Snowflake, Redshift) настолько мощные, что трансформацию проще делать уже после загрузки — это подход ELT.

## Что проектирует аналитик в ETL

| Артефакт | Что описать |
|----------|------------|
| **Источники данных** | Какие системы-источники, формат, частота обновления, API или БД |
| **Маппинг полей** | Соответствие полей источника → полям целевой системы |
| **Правила трансформации** | Формулы, фильтры, преобразования типов, обработка NULL |
| **Расписание** | Частота загрузки: раз в час, раз в день, real-time |
| **SLA** | Когда данные должны быть доступны (например, к 9:00 утра) |
| **Обработка ошибок** | Что делать при ошибке: retry, alert, пропуск записи |
| **Data Quality** | Проверки качества: полнота, уникальность, допустимые диапазоны |

## Пример маппинга полей (из практики)

| Источник (CRM) | Поле CRM | Целевая таблица DWH | Поле DWH | Трансформация |
|---|---|---|---|---|
| Таблица `clients` | `full_name` | `dim_customer` | `first_name`, `last_name` | Разделить по первому пробелу |
| Таблица `clients` | `phone` | `dim_customer` | `phone_normalized` | Удалить пробелы, привести к +7... |
| Таблица `orders` | `created_date` | `fact_orders` | `order_date` | Конвертировать из DD.MM.YYYY в DATE |
| Таблица `orders` | `amount` | `fact_orders` | `amount_rub` | Умножить на курс, если валюта не RUB |

## Популярные ETL/ELT-инструменты

| Инструмент | Тип | Для кого |
|------------|-----|----------|
| **Apache Airflow** | Оркестратор | Инженеры данных (Python) |
| **dbt** | Трансформация (ELT) | Аналитики (SQL) |
| **Apache NiFi** | ETL с UI | Low-code интеграции |
| **Talend** | ETL-платформа | Enterprise |
| **Fivetran / Airbyte** | Извлечение (EL) | Быстрое подключение источников |
| **Apache Spark** | Big Data ETL | Большие объёмы данных |

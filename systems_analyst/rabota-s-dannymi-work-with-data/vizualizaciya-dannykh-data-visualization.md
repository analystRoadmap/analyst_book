# Визуализация данных (Data Visualization)

| Грейд | Роль |
|-------|------|
| 🔵 Middle | 🅱️🅰️ BA + SA |

> Визуализация — это не «нарисовать красивый график», а превратить данные в решение. Аналитик должен уметь выбрать правильный тип визуализации, построить осмысленный дашборд и донести инсайты до бизнеса.

## Зачем аналитику визуализация

Визуализация — мост между сырыми данными и бизнес-решениями. Таблица с 10 000 строк ничего не скажет директору. А график, показывающий падение конверсии на 15% после последнего релиза — скажет.

Типичные задачи аналитика:
- Построить дашборд для мониторинга ключевых метрик (KPI)
- Подготовить визуализацию для презентации результатов
- Исследовать данные (exploratory data analysis) для поиска гипотез
- Настроить алерты при отклонении метрик от нормы

## Какой график для какой задачи

| Тип визуализации | Когда использовать | Пример |
|------------------|-------------------|--------|
| **Line chart** | Тренды во времени | Продажи по месяцам, DAU по дням |
| **Bar chart** | Сравнение категорий | Выручка по регионам, топ-10 товаров |
| **Pie chart** | Доли целого (2–5 сегментов!) | Структура выручки по каналам |
| **Scatter plot** | Корреляция двух переменных | Зависимость конверсии от времени на сайте |
| **Heatmap** | Плотность/интенсивность | Активность пользователей по часам и дням |
| **Funnel** | Этапы конверсии | Регистрация → Активация → Оплата |
| **Table** | Детальные данные | Список заказов с фильтрами |
| **KPI card** | Одно ключевое число | Выручка за день: ₽1.2M (+5%) |

**Правило:** если pie chart содержит больше 5 сегментов — замените на bar chart. Если line chart содержит больше 7 линий — упростите.

## Инструменты визуализации

### Для бизнес-пользователей и аналитиков (no-code/low-code)

| Инструмент | Особенности | Стоимость |
|------------|------------|-----------|
| **Tableau** | Мощный, интерактивный, лидер рынка | Платный (есть Tableau Public бесплатно) |
| **Power BI** | Интеграция с Microsoft, DAX | Бесплатная десктоп-версия |
| **Looker (Google)** | LookML, интеграция с BigQuery | Платный (часть Google Cloud) |
| **Metabase** | Open Source, простой в настройке | Бесплатный / Cloud платный |
| **Apache Superset** | Open Source, мощный | Бесплатный |
| **Google Data Studio (Looker Studio)** | Бесплатный, интеграция с Google | Бесплатный |

### Для технических дашбордов

| Инструмент | Для чего |
|------------|----------|
| **Grafana** | Мониторинг инфраструктуры, метрики, логи |
| **Kibana** | Визуализация данных из Elasticsearch |
| **Redash** | SQL-based дашборды, Open Source |

### Для программистов и data-аналитиков

| Инструмент | Язык |
|------------|------|
| **Matplotlib / Seaborn** | Python |
| **Plotly** | Python / JavaScript |
| **ggplot2** | R |
| **D3.js** | JavaScript (кастомные визуализации) |

## Принципы хорошего дашборда

1. **Один дашборд — одна аудитория.** Дашборд для CEO и дашборд для аналитика — разные вещи.
2. **Пирамида информации:** наверху — KPI-карточки (выручка, конверсия), ниже — графики трендов, внизу — детальные таблицы.
3. **Не больше 6–8 виджетов** на один экран. Перегруженный дашборд никто не читает.
4. **Фильтры и drill-down.** Дайте пользователю возможность переключать период, регион, сегмент.
5. **Подписи и контекст.** Число «₽1.2M» без сравнения бессмысленно. Добавьте: «vs ₽1.1M в прошлом месяце (+9%)».
